What is CSS?

CSS stands for Cascading Style Sheets. It is the language for describing the presentation of Web pages, including colours, layout, and fonts, thus making our web pages presentable to the users.

CSS is designed to make style sheets for the web. It is independent of HTML and can be used with any XML-based markup language. Now let’s try to break the acronym:

  • Cascading: Falling of Styles
  • Style: Adding designs/Styling our HTML tags
  • Sheets: Writing our style in different documents

CSS Structure and Syntax

The Structure of CSS

CSS is composed of rules that define how HTML elements should be displayed. A CSS rule consists of a selector and a declaration block:

  • Selector: The HTML element(s) you want to style. It can be a single element, a class, an ID, or even a group of elements.
  • Declaration Block: Contains one or more declarations, each of which specifies a property and a value (e.g., color: blue;). The declaration block is enclosed in curly braces {}.
  • There are three basic selectors namely: tags, id’s, and classes
  • Selector: selects the element you want to target
  • Property: name(attributes) like color, font-size, background, width, height,etc
  • Value: values associated with these properties
        
          Selector {
          Property(name) 1 : value;
          Property(name) 2 : value;
          Property(name) 3 : value;
          }

          For example:

          h1
          {
          Color: red;
          Text-align: center;
          }

          #unique 
          {
          color: green;
          }

          .section: {
            font-size: 16px;
          }

CSS How-To

  • There are 3 ways to write CSS in our HTML file.
    • Inline CSS
    • Internal CSS
    • External CSS
  • Priority order
    • Inline > Internal > External

Inline CSS

This style is applied directly within an HTML element using the style attribute. It’s useful for small, specific changes but can make the HTML code cluttered.

  • Before CSS this was the only way to apply styles
  • Not an efficient way to write as it has a lot of redundancy
  • Self-contained
  • Uniquely applied on each element
  • The idea of separation of concerns was lost
Example in code: 


<h3 style="color:red"> Have a great day </h3>
<p style="color: green"> I did this , I did that </p>
Output in browser:

Have a great day

I did this, I did that

  Internal CSS

Placed within the style tag inside the head section(headtag) of an HTML document, internal CSS allows you to define styles for a single page, keeping the styling separate from the content but still within the same file.

  • With the help of style tag, we can apply styles within the HTML file
  •  Redundancy is removed
  • But the idea of separation of concerns still lost
  • Uniquely applied on a single document
Example in code: 
    < style>
h1{
 color:red;
}
</style>  
<h1> Have a great day </h1>
Output in browser:

Have a great day

  External CSS

This method uses a separate .css file linked to the HTML document via the <link /> tag. It’s ideal for maintaining consistent styling across multiple pages, enabling easier updates and cleaner code management.

  • With the help of <link> tag in the head tag, we can apply styles
  • Reference is added 
  • File saved with .css extension
  • Redundancy is removed
  • The idea of separation of concerns is maintained
  • Uniquely applied to each document
Example in code: 

(.html file) 
<head>
<link rel="stylesheet" type="text/css" href="name of the Css file">
</head> (passing the .css file link)

in the world.</span>

  color:blue;    
  font-size: 15px;

Now that we have seen all the three selectors now let’s see how style falls or cascades. We will implement one program where we add style on the same element by using a tag, id’s, and classes as selectors. The objective of this is to show how one style cuts the other style that might also be referred to as Priority. We will see that id have the highest priority over tags and classes.

Now let’s see its implementation:

<!DOCTYPE html>
<html>
<head>
<title>HTML</title>
<style>

p{
color:red;
}
.class {
color:green;
}
#id{
color:orange;
}

</style>
</head>
<body>

<p id="id" class="class"> This is CSS </p> 

</body>
</html>

If you have observed how one style is fighting against another in order to style the element. Here the race was won by id, what if all the selectors are classes or tags then the one which is closer or applied at the end will win the race and what if a class and tag selectors are used on the same element, in that case, the race will be won by the class selector.

Note

  • Element Selector: Targets HTML elements by their tag name (e.g., p, h1)
  • Class Selector: Targets elements with a specific class attribute. Classes are defined with a period (.) followed by the class name (e.g., .example).
  • ID Selector: Targets a single element with a specific ID. IDs are defined with a hash (#) followed by the ID name (e.g., #header).
  • Attribute Selector: Targets elements based on attributes and their values (e.g., [type="text"]).
  • Pseudo-class Selector: Targets elements in a specific state (e.g., :hover, :focus).
  • .Pseudo-element Selector: Targets a specific part of an element (e.g., ::before, ::after).

CSS Colors

  • There are different colouring schemes in CSS
  • Three widely used techniques are as follows
    • RGB Color Scheme
      • This starts with RGB and takes 3 parameter
      • 3 parameter basically corresponds to red, green and blue
      • The value of each parameter may vary from 0 to 255.
        • Eg: RGB(255,0,0); means color red
        • Eg: RGB(255,255,255); means color white
        • Eg: RGB(0,0,0); means color black
    • HEX Color Scheme
      • Hex code starts with # and comprises of 6 numbers which are further divided into 3 sets
      • Sets basically correspond to Red, Green, and Blue
      • Single set value can vary from 00 to 09 and AA to FF
      • Hex color code is the most widely used color code scheme
        • Eg: #ff0000 ; means color red
        • Eg: #fffff ; means color white
        • Eg: #0000 ; means color black
    • RGBA Color Scheme
      • This starts with RGB and takes 4 parameter
      • 4 parameter basically corresponds to red, green, blue and alpha(transparent)
      • Value of the first three parameters may vary from 0 to 255 and the last parameter ranges from 0 to 1 that is from 0.1, 0.2,…..0.9
      • Eg: RGB(255,0,0,0); means color red

CSS Size Units

CSS sizes are used to define the dimensions, spacing and size of an element on a webpage, such as the width, height, padding, margins, and fonts sizes on a webpage. Using different size units effectively is crucial for responsive layouts, with CSS offering various units for different needs.

  1. Absolute Units
    Absolute units have fixed values, meaning they represent the same size regardless of the context. These units are not affected by the size of other elements or the viewport.

    Type of absolute units size

    • Pixels (px): Pixels are the most commonly used absolute unit in CSS. One pixel is generally considered as one dot on the screen. Pixels are useful for precise control, but they do not scale well on different screen sizes.
      Example: width: 200px;
  2. Relative Units
    Relative units are dynamic and scale in relation to other elements or the viewport. They are essential for creating responsive designs that adapt to different screen

    Type of relative units size

    • Em (em):The em unit is relative to the font size of the parent element. If the parent element has a font size of 16px, 1em equals 16px. Em is commonly used for font sizes, padding, and margins because it scales with the parent’s size.
      Example: font-size: 1.5em; (1.5 times the parent’s font size)
    • Rem (rem): The rem unit is relative to the root element’s font size (usually the html tag element). Unlike em, which scales based on its nearest parent, rem scales based on the root element, providing more consistency across elements.
      Example: margin: 2rem; (2 times the root’s font size)
    • Percentage (%): Percentages are relative to the size of the parent element. For example, width: 50% will make the element half the width of its parent. Percentages are commonly used for widths, heights, padding, and margins.
      Example: width: 100%; (full width of the parent)

CSS BoxModel

In CSS, every element is structured according to the Box Model, which is a fundamental concept for controlling layout and spacing. The Box Model defines how elements are displayed on a webpage and consists of four key areas:

  1. Content Edge: This is the innermost part of the box and contains the actual content, such as text or images.
  2. Padding Edge: Surrounding the content is the padding, which adds space between the content and the border. This space is inside the element's border.
  3. Border Edge: The border wraps around the padding and content. It can be styled with different widths, colors, and patterns.
  4. Margin Edge: The margin is the outermost layer, outside the border, and it controls the space between the element and neighboring elements.

When you inspect an element in Chrome (right-click > Inspect), the browser visually displays the Box Model on the right side. Here, you can see how content, padding, border, and margin relate to each other. The innermost rectangle represents the content, followed by the padding, then the border, and finally the margin on the outside. This visualization helps developers understand and adjust the layout and spacing of elements on the page.

Css Flexbox

Overview

Flexbox is a one-dimensional layout method for arranging items in a row or column. It provides better control over the alignment, direction, and order of elements within a container, making it perfect for building responsive web designs.

Key Concepts

  • Flex Container: The parent element that holds flex items. It is defined using display: flex;.
  • Flex Items: The children of the flex container.

Main Properties

  1. Container Properties
    • display: flex; - Defines the flex container.
    • flex-direction: Defines the direction of the flex items (row, row-reverse, column, column-reverse).
    • justify-content: Aligns items horizontally (flex-start, flex-end, center, space-between, space-around, space-evenly).
    • align-items: Aligns items vertically (flex-start, flex-end, center, baseline, stretch).
    • flex-wrap: Allows items to wrap onto multiple lines (nowrap, wrap, wrap-reverse).
    • gap: The gap property in CSS Flexbox (and Grid) is used to define the space between items within a flex container. It’s a shorthand for setting both row-gap and column-gap and simplifies spacing between flex items without needing to add margin or padding to individual items.
  2. Item Properties
    • flex: A shorthand for flex-grow, flex-shrink, and flex-basis.
    • align-self: Allows an item to override the container’s align-items property.
    • order: Controls the order of items.

Example: Basic Flexbox Layout in Css

<!DOCTYPE html>
html code------------------- ⇓
<div class="flex-container">
  <div class="flex-item">Item 1</div> 
  <div class="flex-item">Item 2</div>
  <div class="flex-item">Item 3</div>
</div> 

css code--------------------⇓
.flex-container {
  display: flex;
  flex-direction: row; /* Aligns items horizontally */
  justify-content: space-between; /* Space between items */
  align-items: center; /* Aligns items vertically */
  height: 200px;
}

.flex-item {
  background-color: lightcoral;
  padding: 20px;
  margin: 5px;
  color: white;
  text-align: center;
  flex: 1; /* Each item takes equal space */
}

Flexbox in Action

  • Horizontal Alignment: By changing justify-content, you can align items to the left, right, or center, or distribute space between them.
  • Vertical Alignment: Using align-items, you can align items at the top, center, or bottom of the container.
  • Responsive Design: Flexbox allows items to wrap and adjust their size, making it perfect for responsive layouts.

Css Grid Layout

Overview

Grid Layout is a two-dimensional layout system, meaning it can handle both rows and columns. It provides more precise control over the placement of items on a page, making it ideal for complex designs.

Key Concepts

  • Grid Container The parent element that holds grid items. It is defined using "display: grid;".
  • Grid Items: The children of the grid container.

Main Properties

  1. Container Properties
    • display: grid; - Defines the grid container.
    • grid-template-columns: Defines the number and size of columns.
    • grid-template-rows: Defines the number and size of rows.
    • grid-gap: Defines the space between grid items (row-gap and column-gap).
    • justify-items: Aligns items horizontally within their grid area.
    • align-items: Aligns items vertically within their grid area.
  2. Item Properties
    • grid-column: Specifies which columns an item should span.
    • grid-row: Specifies which rows an item should span.
    • justify-selfAligns an individual item horizontally.
    • align-selfAligns an individual item vertically.

  text-align: center;

Css Positioning

Overview

CSS positioning is a key aspect of web design that determines how elements are placed on a web page. Understanding positioning helps you control the layout and flow of elements, allowing for more precise and creative designs.

There are several positioning schemes in CSS, each serving a unique purpose. Here's everything you need to know about CSS positioning:

Types of CSS Positioning

  1. Static Positioning
    • Default Positioning: Every HTML element is positioned static by default. This means elements are placed in the normal document flow, one after the other, as they appear in the HTML.
    • No Control: static positioning doesn’t allow any control over the element's exact position on the page. Top, right, bottom, and left properties don’t apply to statically positioned elements.
    • Typically used when no specific positioning is needed, and elements should flow naturally within the document. 
    div {
    position: static; /* This is the default */ 
   }
  2. Relative Positioning
    • Position Relative to Itself: An element with position: relative; is positioned relative to its normal position in the document flow. The top, right, bottom, and left properties can be used to adjust its position.
    • Doesn’t Affect Other Elements: Moving an element with relative positioning doesn't affect the position of surrounding elements.
    • position: relative; top: 10px; /* Moves the element 10px down */ left: 20px; /* Moves the element 20px to the right */ Adjusting Position Slightly: Useful when you need to nudge an element without disrupting the flow of the document. 
      • div {
      position: relative;
      top: 10px;  /* Moves the element 10px down */
      left: 20px; /* Moves the element 20px to the right */
   }
  3. Absolute Positioning
    • Position Relative to the Nearest Positioned Ancestor: An element with position: absolute; is removed from the normal document flow and positioned relative to its nearest positioned ancestor (any parent with position: relative;, absolute;, or fixed;). If no such ancestor exists, it will be positioned relative to the initial containing block (usually the viewport).
    • Free Placement: You can place the element anywhere using top, right, bottom, and left properties.
    • Overlapping Elements: Perfect for creating overlays, tooltips, dropdowns, or any elements that need to be positioned independently of the document flow. 
      • .container{
     position: relative; /* Positioned ancestor */
      }

.abs-element {
  position: absolute;
  top: 50px;  /* 50px from the top of the container */
  right: 20px; /* 20px from the right of the container */
}
  4. Fixed Positioning
    • Fixed to the Viewport: An element with position: fixed; is positioned relative to the viewport, not any ancestor. This means it stays fixed in place even when the page is scrolled.
    • Useful for Sticky Elements: Fixed elements stay in the same position regardless of scroll, making them ideal for sticky headers, footers, or floating action buttons.
    • Sticky Headers/Footers: Often used for elements that need to remain visible while scrolling, such as navigation bars, back-to-top buttons, or persistent sidebars. 
      • .fixed-element {
        position: fixed;
        top: 0; /* Sticks to the top of the viewport */
        left: 0; /* Sticks to the left of the viewport */
        width: 100%;
        background-color: #333;
      }
  5. Sticky Positioning
    • Hybrid of Relative and Fixed: position: sticky; is a mix of relative and fixed positioning. An element with sticky positioning behaves like a relative element until it reaches a specified point (determined by top, right, bottom, or left), at which point it becomes fixed.
    • Scroll-based Positioning: Sticky elements are great for creating elements that stay within a section until a certain scroll position is reached.
    • Interactive Content: Useful for creating sticky headers that follow you down the page until the next section, or for sticky table headers that remain in view while scrolling. 
      • .sticky-element {
        position: sticky;
        top: 10px; /* Becomes fixed 10px from the top of the viewport */
        background-color: yellow;
      }

Understanding Z-Index in Css Positioning

The z-index property in CSS is used in conjunction with positioning to control the stacking order of elements on a web page. It determines which elements appear on top of others when they overlap.

  • Stacking Order: z-index controls the stacking order of positioned elements (elements with position: relative;, absolute;, fixed;, or sticky;). Elements with a higher z-index appear in front of elements with a lower z-index.
  • Only Applies to Positioned Elements: It has no effect on elements with position: static;.
  • Layering Elements: Essential for creating complex layouts where elements overlap, such as pop-ups, modals, or layered backgrounds. 

.element1 {
position: absolute;
z-index: 1; /* Behind element2 */
background-color: red;
}

.element2 {
position: absolute;
z-index: 2; /* In front of element1 */
background-color: blue;
}

In this example, element2 will appear on top of element1 because it has a higher z-index value.

Css Box Shadow

Overview

The box-shadow property in CSS allows you to add shadows to elements, giving them depth and a more layered, realistic appearance. This property is highly customizable, enabling you to create a variety of shadow effects, from subtle to dramatic.

Basic Syntax

The box-shadow property is defined as follows:

<!DOCTYPE html>
              box-shadow: offset-x offset-y blur-radius spread-radius color;

Main Properties

  1. box-shadow: offset-x offset-y blur-radius spread-radius color;
  2. offset-y: The vertical distance of the shadow from the element. A positive value moves the shadow down, while a negative value moves it up.
  3. blur-radius (optional): Defines the blur radius of the shadow. A larger value makes the shadow more blurred, while a value of 0 makes it sharp.
  4. spread-radius (optional): Controls the size of the shadow. Positive values increase the size, while negative values decrease it.
  5. color: Defines the color of the shadow. You can use any valid CSS color format (e.g., named colors, hex, rgba, etc.).

Example: Basic Box Shadow

<!DOCTYPE html>
html code------------------- ⇓
<div class="box"></div> 

css code--------------------⇓
.box {
  width: 100px;
  height: 100px;
  background-color: lightblue;
  box-shadow: 10px 10px 5px #888888; /* Horizontal 10px, Vertical 10px, Blur 5px, Color #888888 */
}
Output in browser:

Explanation:

  • Offset-x: 10px (shadow moves 10px to the right).
  • Offset-y: 10px (shadow moves 10px down).
  • Blur-radius:#888888 (a medium gray).
  • Color:#888888 (a medium gray).

The above code creates a light blue box with a shadow that is 10px to the right and down from the box, with a slight blur.

Css Transitions

Overview

CSS Transitions allow you to change property values smoothly over a specified duration, providing an engaging visual experience to users. Instead of abruptly switching from one style to another, transitions enable a gradual transformation, making interactions and animations more polished.

What Are CSS Transitions?

CSS Transitions let you define the transition effect that should be applied when a CSS property changes. This can be triggered by various events, such as hovering over an element, clicking it, or changing the class of an element. 


transition: property duration timing-function delay;

Main Properties

  1. property: The name of the CSS property you want to animate (e.g., width, height, color). You can use all to apply the transition to all animatable properties.
  2. duration: The time it takes for the transition to complete (e.g., 1s, 500ms).
  3. timing-function: Describes how the intermediate states of the transition are calculated. Common values include ease, linear, ease-in, ease-out, and ease-in-out.
  4. delay (optional): The time to wait before starting the transition.

Example: Simple Transition on Hover

  border: none;
  cursor: pointer;
  transition: background-color 0.5s ease, transform 0.3s ease-in-out;
}

.btn-transition:hover {
  background-color: #2ecc71;
  transform: scale(1.1);
}
Output in browser:

Explanation:

  • Property: background-color and transform are the properties being transitioned.
  • Duration: 0.5s for background-color, 0.3s for transform.
  • Timing-function: ease for a smooth start and end, ease-in-out for the scale effect.
  • Effect: When the button is hovered, its background color changes from blue to green, and it slightly enlarges.

Common Timing Functions In Css Transition

  • Ease: Starts slow, speeds up, then slows down.

  • ease-in Starts slow and then speeds up.

  • ease-in-out Combines ease-in and ease-out for a smooth start and finish.

Css Animations

Overview

CSS animations allow you to animate transitions between CSS property values without JavaScript. Animations are controlled using keyframes and properties such as animation-duration, animation-timing-function, and more. Here's a more detailed breakdown of CSS animations.

Key Components of CSS Animation

  1. @keyframes Rule: The @keyframes rule specifies the intermediate steps (or "keyframes") in the animation sequence. Each keyframe defines a specific point in the animation sequence by using percentages (from 0% to 100%) or the keywords from (equivalent to 0%) and to (equivalent to 100%). Inside each keyframe, you can specify which CSS properties you want to animate. CSS properties that can be animated include colors, dimensions, positioning, opacity, and more. 
    
  css code--------------------⇓
  @keyframes animation-name {
    0% {
      /* Initial styles */
    }
    50% {
      /* Midway styles */
    }
    100% {
      /* Final styles */
    }
  }
  2. CSS Animation Properties: After defining the keyframes, you apply them to an element using the following properties:
    • animation-name: Specifies the name of the keyframes to be applied.
    • animation-duration: Defines how long the animation should take to complete one cycle (e.g., 2s for 2 seconds).
    • animation-timing-function: Describes how the animation progresses over time (e.g., linear, ease, ease-in, etc.).
    • animation-delay: Specifies a delay before the animation starts.
    • animation-iteration-count: Defines how many times the animation will play (e.g., infinite for looping forever).
    • animation-direction: Specifies whether the animation should play in reverse on alternate cycles (normal, reverse, alternate, alternate-reverse).
    • animation-fill-mode: Defines what styles are applied before/after the animation.
    • animation-play-state: whether the animation is running or paused.

Example with Explanations

Here's a more advanced example to showcase all of these features in use: 



html code------------------- ⇓
<div class="animated-box"></div> 

css code--------------------⇓
@keyframes slideAndFade {

  }
  50% {
    transform: translateX(200px); /* Move to the right */
    opacity: 0.5; /* Half opacity */
    background-color: yellow;
  }
  100% {
    transform: translateX(0); /* Move back to the original position */
    opacity: 1; /* Fully visible */
    background-color: green;
  }

  position: relative;
  width: 100px;
  height: 100px;

  animation-name: slideAndFade;

  animation-delay: 1s;
  animation-iteration-count: infinite;
  animation-direction: alternate;
  animation-fill-mode: forwards;
}

Animation Timing Function

The animation-timing-function controls the speed curve of the animation. Common values include:

  • linear: The animation has a constant speed.
  • ease: The default value, which starts slowly, accelerates, and then slows down.
  • ease-in: The animation starts slow, then speeds up.
  • ease-out: The animation starts fast and then slows down.
  • ease-in-out: The animation starts and ends slowly, with a faster middle section.

Css Media Queries

CSS Media Queries are a powerful tool that allows you to apply different styles based on the characteristics of the device or browser window. Media queries enable responsive design, which helps websites adapt to various screen sizes, orientations, and resolutions.
This makes it easy to ensure a website looks good on all devices, from large desktop monitors to small mobile phones.

Basic Syntax


css code--------------------⇓
@media (condition) {
  /* CSS rules go here */
}

You specify the condition within parentheses. If that condition is true (e.g., the screen width is less than 600px), the styles inside the media query will be applied.

Common Media Query Conditions

  • width / min-width / max-width:The width of the browser window.
  • height / min-height / max-height: The height of the browser window.
  • orientationCan be either landscape or portrait. up.

Example in Code

Here’s an example to demonstrate how media queries work in a responsive design. 


css code--------------------⇓
body {
  font-family: Arial, sans-serif;
  padding: 20px;
}

.responsive-box {
  width: 100%;
  padding: 20px;
  color: white;
  text-align: center;
}

/* Default styles for large screens */
.responsive-box {
  background-color: blue;
}

/* Styles for screens smaller than 768px (tablets and smaller) */
@media (max-width: 768px) {
  .responsive-box {
    background-color: green;
  }
}

/* Styles for screens smaller than 480px (mobile phones) */
@media (max-width: 480px) {
  .responsive-box {
    background-color: red;
  }
}
